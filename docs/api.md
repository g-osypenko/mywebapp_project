# Документація API (Simple Inventory)

Усі бізнес-ендпоінти підтримують Content Negotiation. Формат відповіді залежить від HTTP-заголовка `Accept`:

* `Accept: text/html` — повертає HTML.
* `Accept: application/json` або `*/*` — повертає JSON.

## Ендпоінти бізнес-логіки

### `GET /items`

Отримання списку всіх предметів в інвентарі.

* **Відповідь:** Масив об'єктів (або HTML таблиця) з полями `id`, `name`.

### `POST /items`

Створення нового запису в інвентарі.

* **Тіло запиту (JSON):** `{"name": "Назва", "quantity": 10}`
* **Відповідь (201 Created):** `{"message": "Item created successfully", "id": 1}`

### `GET /items/{id}`

Отримання детальної інформації про предмет.

* **Відповідь:** Об'єкт (або HTML сторінка) з полями `id`, `name`, `quantity`, `created_at`.
* **Помилки:** `404 Not Found`, якщо запис відсутній.

## Системні ендпоінти

### `GET /`

Кореневий ендпоінт.

* **Відповідь:** Завжди віддає `text/html` зі списком усіх ендпоінтів бізнес-логіки.

### `GET /health/alive`

Перевірка життєздатності процесу.

* **Відповідь:** `200 OK` (plain text).

### `GET /health/ready`

Перевірка стану підключення до MariaDB.

* **Відповідь:** `200 OK`, якщо підключення успішне. `500 Internal Server Error`, якщо з'єднання з БД відсутнє.
