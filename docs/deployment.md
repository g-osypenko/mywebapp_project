# Документація з розгортання та тестування

## Вимоги до середовища

* **ОС:** Ubuntu Server 22.04 LTS .
* **CPU:** 1-2 vCPU.
* **RAM:** Мінімум 1 GB (рекомендовано 2 GB для стабільної роботи MariaDB).
* **Disk:** Мінімум 10 GB.
* **Спеціальні налаштування під час інсталяції:** Встановити OpenSSH Server для віддаленого доступу.

## Доступ до ВМ

* **Підключення:** `ssh user@<ip_address>` .

## Інструкція з розгортання

1. Скопіюйте директорію проєкту `mywebapp_project` на віртуальну машину (наприклад, через `scp` або `git clone`).
2. Перейдіть у директорію проєкту: `cd mywebapp_project`
3. Запустіть скрипт автоматизації від імені root:
   ```
   sudo bash scripts/deploy.sh
   ```
4. Скрипт автоматично встановить залежності, налаштує MariaDB, Nginx, UFW, створить системних користувачів та запустить Systemd-сервіс.

## Облікові записи (створюються автоматично)

* `student` / пароль: `12345678` вимагає зміни при першому вході, має sudo
* `teacher` / пароль: `12345678` вимагає зміни при першому вході, має sudo
* `operator` / пароль: `12345678` вимагає зміни при першому вході, має обмежений sudo для systemctl/nginx
* `app` системний користувач без доступу до shell

## Інструкція з тестування

Після успішного розгортання виконайте наступні перевірки:

1. **Перевірка Nginx (Reverse Proxy):**
   Відкрийте у браузері `http://<ip_address_vm>/`. Має відобразитися HTML-сторінка зі списком ендпоінтів.
2. **Перевірка блокування портів (UFW):**
   Спробуйте доступитися напряму до порту застосунку: `curl http://<ip_address_vm>:8080/`. Запит має завершитися таймаутом (порт закритий ззовні).
3. **Перевірка блокування системних ендпоінтів:**
   Запит `curl http://<ip_address_vm>/health/alive` має повернути `404 Not Found` (обмеження Nginx).
4. **Створення запису (POST):**
   ```
   curl -X POST http://<ip_address_vm>/items -H "Content-Type: application/json" -d '{"name": "Test Item", "quantity": 5}'
   ```
5. **Отримання даних (Content Negotiation):**
   * JSON: `curl -H "Accept: application/json" http://<ip_address_vm>/items`
   * HTML: `curl -H "Accept: text/html" http://<ip_address_vm>/items`
